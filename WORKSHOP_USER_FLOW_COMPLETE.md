┌─────────────────────────────────────────────────────────────┐

        │                     │

### 핵심 차이점

| 구분 | Version 1 (기존) | Version 2 (5-Phase) |
|------|-----------------|---------------------|
| **초점** | 칸반보드 + 업무 추출 | 전략 갭 발견 + AI 자동화 |
| **워크플로우** | 6 Steps | 5 Phases |
| **시간** | ~30분 | ~75분 |
| **결과물** | 칸반보드, AI 분석 | 실행 가능한 코드/프롬프트 패키지 |
| **구현 상태** | 프로덕션 가능 | 데모 완성, 백엔드 미구현 |
| **기술** | Next.js + TypeScript | HTML 데모 (프로토타입) |

  }

    default: return '👤';
  }

const getSummary = () => {
  const autoSteps = steps.filter(s => s.role === 'llm').length;
  const augmentedSteps = steps.filter(s => s.role === 'copilot').length;

## 데이터 흐름

### 버전 1 플로우

```
사용자 → 프론트엔드 → 백엔드 API → Claude AI → 데이터베이스
                 ↓
            localStorage (임시)
```

**Step별 데이터**:
1. Step 1: domains → localStorage → DB
2. Step 2: columns → localStorage → DB
3. Step 3: 선택 → 라우팅
4. Step 4: files → S3, analysis → AI, tasks → DB
5. Step 5: proposals → AI 생성
6. Step 6: workflows → AI 생성, packages → ZIP
7. Kanban: 모든 데이터 통합 표시

### 버전 2 플로우

```
사용자 → 단일 HTML → sessionStorage
                 ↓
          시뮬레이션 데이터
```

**Phase별 데이터**:
- Phase 0: 없음
- Phase 1: context.json → sessionStorage
- Phase 2: current_state.json → sessionStorage
- Phase 3: strategic_gaps.json → sessionStorage
- Phase 4: automation_plan.json → sessionStorage
- Phase 5: workflows → 메모리, packages → 가상
- Final: 전체 통합
