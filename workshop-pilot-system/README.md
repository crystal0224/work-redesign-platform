# 🎯 Work Redesign Platform - 실제 파일럿 테스팅 시스템

## 목적

**30명의 Synthetic Users로 실제 HRD 파일럿 테스팅 프로세스를 완전히 재현**하여:
- ✅ 직무별, 성숙도별, 팀 규모별로 **어느 단계에서 막히는지** 파악
- ✅ **실제로 하기 힘든 부분** 사전 발견
- ✅ **추가 가이드가 필요한 지점** 식별

실제 30명을 불러 테스트하는 것과 동일한 인사이트를 40분 만에 얻습니다.

---

## 🤖 핵심 개념: Synthetic Users

### Synthetic User란?

**Synthetic User** (합성 사용자)는 실제 사용자의 특성, 행동, 의사결정 패턴을 데이터 기반으로 정교하게 모델링한 AI 에이전트입니다.

### 왜 Synthetic Users를 사용하는가?

| 항목 | 실제 사용자 테스트 | Synthetic Users 기반 시뮬레이션 |
|------|-------------------|-------------------------------|
| **소요 시간** | 3일 (모집 + 실행 + 분석) | 40분 (30명 병렬 실행) |
| **비용** | $3,000+ (참가비 + 시설 + 퍼실리테이터) | $1.50 (API 비용만) |
| **반복 가능성** | 1-2회 (예산 제약) | 무제한 (UI 변경 시마다) |
| **세그먼트 통제** | 어려움 (모집 제약) | 정밀 통제 (직무/성숙도/팀 규모) |
| **일관성** | 낮음 (당일 컨디션 영향) | 높음 (동일 조건 재현) |

### 이 시스템의 Synthetic Users

**30명의 Synthetic Users (Personas v3)**:
- ✅ **실제 SK그룹 팀장 프로필 기반** (8개 부서, 4단계 디지털 성숙도)
- ✅ **300+ 데이터 포인트**: 팀 구성, 업무 특성, pain points, 사용 도구, 성격
- ✅ **AI 페르소나 시뮬레이션**: Claude Sonnet/Haiku로 실제 워크샵 참여 재현
- ✅ **Enhanced Prompting**: System identity + Chain-of-Thought + Few-shot examples

**동작 방식**:
```
1. 사전 인터뷰: "김지훈 팀장 (마케팅, Advanced)"으로서 기대/우려 표현
2. 워크샵 진행: 실제 브라우저에서 각 단계 수행 + 막힘 지점 관찰
3. 중간 체크인: 단계별 만족도, 어려움, 계속 여부 판단
4. 사후 인터뷰: 전체 경험 회고 + 개선사항 제안
```

**검증된 신뢰성**:
- 실제 HRD 파일럿 테스트 방법론 재현
- 부서별, 성숙도별 차별화된 피드백 (generic 답변 방지)
- 팀장 관점 일관성 (팀원 워크샵 불참 반영)

## 🚀 실행 방법

### ⭐ 추천: 실제 파일럿 테스팅

```bash
# 5명 샘플 테스트
npm run pilot:real

# 30명 전체 테스트 (병렬 실행)
npm run pilot:real:full

# UI/UX 분석 보고서 생성
npm run pilot:report:uiux

# UI 재추출 (프론트엔드 변경 시)
npm run pilot:ui-extract
```

**실행 단계:**
1. **사전 인터뷰**: 참가자 배경, 기대, 우려 파악
2. **워크샵 진행**: Playwright로 실제 브라우저 실행 + 관찰
3. **중간 체크인**: 각 단계 후 "지금까지 어떠세요?"
4. **사후 인터뷰**: 전체 경험 회고
5. **퍼실리테이터 분석**: 막힘 지점, 이탈 위험, 개선사항 도출

**소요 시간:** 5명 기준 약 15-20분

## 🎯 최신 시스템 특징

### ⭐ 실제 파일럿 프로세스

```
✅ 사전 인터뷰로 참가자 우려 파악
✅ 실제 브라우저에서 Playwright 실행
✅ 각 단계별 중간 체크인 ("지금까지 어떠세요?")
✅ 실시간 관찰 (막힘, 에러, 시간 초과, 이탈 위험)
✅ 사후 인터뷰로 전체 회고
✅ 퍼실리테이터 분석 및 제언
```

### ⭐ Synthetic Users (Personas v3)

**30명의 정교한 Synthetic Users**:
```
✅ 팀장 개인 프로필 (리더십 스타일, 경력, 이전 역할)
✅ 팀 단위 디지털 성숙도 (Beginner→Expert 4단계)
✅ 업무 구조화 정도 (비구조화/반구조화/고도구조화)
✅ 구체적 pain points (예: "주당 8시간 수작업 데이터 통합")
```
