## 페르소나 목록

### P021: 최윤석 - SK하이닉스 재무기획팀 팀장
- **나이**: 38세
- **팀장 경력**: 1.0년 (신임)
- **팀 규모**: 10명 (시니어 4명 + 주니어 6명)
- **디지털 성숙도**: Advanced (팀 전체)

**주요 업무**:
- 연간/분기별 예산 편성 (년 3회 RPA)
- 매월 재무 성과 분석 및 경영진 보고
- 중기경영계획(3년) 수립 및 시나리오 분석
- 투자 사업 타당성 검토
- 이사회/주주총회 재무 자료 준비

**신임 팀장의 주요 고민 (Pain Points)**:
1. 시니어 분석가(12년차)가 본인보다 연차가 높아 데이터 해석에 이견이 있을 때 설득 어려움
2. 8개 캠페인 예산이 동시 진행되는데 우선순위 정하기 어려움
3. 월말 예산 통합, 차트 작성 등 반복 업무의 자동화 필요
4. 시니어 분석가의 "우리 방식"에 새로운 방법 도입 저항
5. 긴급 분석 요청으로 인한 업무 계획 왜곡

**기대 행동**:
- Dropout Risk: 15%
- 문제 단계: [2, 5, 8] - 팀 조율, 시간 압박, 자동화 주제
- 강점 단계: [3, 4, 6, 7, 9] - 분석, 데이터 해석, 도구 활용

### P022: 이재훈 - SK이노베이션 회계팀 팀장
- **나이**: 41세
- **팀장 경력**: 0.7년 (신임)
- **팀 규모**: 13명 (시니어 5명 + 주니어 8명)
- **디지털 성숙도**: Intermediate

**주요 업무**:
- 월간 회계 결산 (매월 중순까지)
- 분기/연간 결산 마감 (분기 20일, 연간 30일 내)
- 회계 전표 검토 및 승인 (일 50-100건)
- 세무 신고 (분기 부가세, 연간 법인세)
- 감사 대응 (외부 회계감사, 세무조사)

**신임 팀장의 주요 고민**:
1. 각 사업부의 전표 형식이 제각각 - 시니어들이 일일이 수정하는데 시간 소요
2. 감사인의 계속되는 질문에 팀장이 다 답변해야 함
3. 주니어 5명의 회계 숙련도가 다양 - 신입 교육 부담 크고 시니어 업무 끌려감
4. SAP와 Excel 동시 사용으로 데이터 불일치 발생
5. 세무 법령 변화 빠르고 외부 세무사와 의견 차이 발생

**기대 행동**:
- Dropout Risk: 25%
- 문제 단계: [2, 5, 8, 9] - 팀 표준화, 시간 부족, 법규 변화 대응
- 강점 단계: [1, 3, 4, 6, 10] - 현황 파악, 업무 분석, 규정 준수

4. 은행 담당자 자주 바뀌어 관계 구축 반복
5. Bloomberg와 회사 자금시스템 데이터 불일치 정산에 시간 소요

**기대 행동**:
- Dropout Risk: 20%
- 문제 단계: [2, 5, 8] - 팀 조율, 시간 압박, 실시간 대응
- 강점 단계: [3, 4, 6, 7, 9, 11] - 분석, 도구 활용, 신속한 의사결정

- 원가 절감 프로젝트 기획 및 실행

5. 원가 절감 프로젝트 기획해도 생산팀 저항 많음

**기대 행동**:
- Dropout Risk: 30% (높음)
- 문제 단계: [2, 5, 8, 9] - 팀 조율, 시간 부족, 복잡성

## 공통 특징 분석

### Finance 팀장들의 공통 Pain Points
1. **데이터 통합/자동화 부족** - Excel 수작업, 여러 시스템 데이터 불일치
2. **시간 압박** - 월말/분기말/연말 극한 상황 (야근 8-20시간/월)
3. **팀 내 역량 불균형** - 시니어와 주니어의 능력 차이 크고 새로운 방식 저항
4. **신임 팀장의 리더십 어려움** - 본인이 분야 전문가인데 팀 조율 미숙
5. **시스템/도구 복잡도** - 여러 ERP/시스템 사용하면서 데이터 동기화 어려움

### 워크샵 반응 예측
- **드롭아웃 리스크**: 15~30% (평균 ~22%)
  - P024 오성택(원가관리팀)이 가장 높음 (30%) - 복잡하고 생산팀 저항 크기 때문
  - P021 최윤석(재무기획팀)이 가장 낮음 (15%) - 데이터 기반 사고, 개선 의지 높음

- **강점 단계**: 주로 [3, 4, 6, 7] - 데이터 분석, 도구 활용, 구체적 전략
- **약점 단계**: 주로 [2, 5, 8] - 팀 조율, 시간 부족, 자동화 설계

### 공통 자동화 필요 영역
1. **데이터 자동 통합** - 여러 시스템 데이터를 한눈에 보기
2. **반복 보고서 자동 생성** - 월간/분기 보고 양식화 및 자동화
3. **의사결정 지원 대시보드** - 실시간 성과 현황 모니터링
4. **프로세스 표준화** - 팀원들의 작업 방식 일관성

## 활용 방안

### 워크샵 커스터마이징
- **P021, P023, P025**: Advanced 디지털 성숙도 + 낮은 드롭아웃 리스크
  → 자동화/AI 도구 활용에 적극적. Step 2, 5, 8에서 팀 특성(재무/자금/경영) 반영한 사례 강조

- **P022, P024**: Intermediate 디지털 성숙도 + 높은 드롭아웃 리스크
  → 기초부터 차근차근 설명 필요. Step 1, 2에서 충분한 시간 할애
  → 기존 시스템(SAP, ERP)과의 연계성 강조하여 저항감 완화

### 팀 특성별 주의사항
| 팀 | 주의사항 |
|----|---------|
| P021 재무기획 | 시니어 분석가의 "정통성" 존중 필요. 새 방법도 "기존 역할 강화"로 프레이밍 |
| P022 회계 | 규정 준수 최우선. 자동화도 "감사 대응 강화"로 설명 |
| P023 자금 | 실시간성 강조. 금융시장 변동성 대응하는 도구로 제시 |
| P024 원가 | 제약 산업 특수성(R&D, 임상) 이해하는 사례 필요 |
| P025 경영관리 | "창의성과 신속성" 해치지 않는 프로세스로 제시 |

## 추가 자료

### Finance 부서 특성 요약
- **재무기획**: 예산/계획 수립 중심 → 데이터 기반 의사결정 강조 필요
- **회계**: 규정 준수 최우선 → 감사/세무 대응 강화로 자동화 정당화
- **자금**: 실시간 유동성 관리 → 신속한 대시보드/알림 시스템 필요
- **원가**: 복잡한 통계/분석 → 신약 개발비 같은 특수 케이스 자동화 중요
- **경영관리**: 전략적 분석 → 시나리오/대시보드 자동화로 의사결정 속도 높임
